# 13. Development roadmap – HES (werknaam)

Status: Definitieve annex
Doel: Gefaseerde realisatie van werkende MVP naar volledig platform.

Webapp-first met notebook-validatie per domein:
- Sprint 0: React + FastAPI basisflow
- Sprint 1: woning + energieprofiel
- Sprint 2: solar + notebook validatie
- Sprint 3: heat pump + notebook validatie
- Sprint 4: EV basis + notebook validatie
- Sprint 5: battery + notebook validatie
- Sprint 6: financial + contract
- Sprint 7: optimization constraints
- Sprint 8+: scenario's, polish, AI uitleg, productisering, integraties

Definition of Done per sprint:
- werkende feature
- testdekking op kernregels
- begrijpelijke output voor eindgebruiker
